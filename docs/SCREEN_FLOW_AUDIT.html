Duplication and Legacy Candidates

Likely Duplications

  • Property flow screens used in Home tab, Properties tab, and onboarding.
  • PropertyInviteAccept appears in both RootNavigator and AuthStack.
  • Multiple auth entry screens (Welcome, Login, SignUp) route to AuthScreen.

Potential Legacy Screens to Verify

  • PropertyPhotos, RoomSelection, RoomPhotography
  • AssetScanning, AssetDetails, AssetPhotos, ReviewSubmit

These should be confirmed by checking active navigation routes and usage in the code.

Recommended Cleanup Path

Phase 1: Freeze Legacy

  • Stop routing to legacy screens in navigation.
  • Document replacements for each legacy screen.

Phase 2: Consolidate

  • Use shared screen groups for the property flow across tabs.
  • Centralize property creation flow into one navigator.

Phase 3: Deprecate

  • Remove unused screens and routes after verification.
  • Update tests and docs.

Phase 4: Simplify

  • Split navigation files by role and flow.
  • Keep RootNavigator as the assembly point.

Immediate Decisions Needed

  • Confirm the single source of truth for property creation (new wizard vs extended legacy flow).
  • Agree on which landlord flow screens are considered legacy.
  • Decide whether the invite acceptance flow should live in Root only or also in Auth.
Once these decisions are locked, deprecation can begin without risk of breaking current flows.
