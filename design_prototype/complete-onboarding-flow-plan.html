üí° Design Rationale & Airbnb Alignment

Why Reuse PropertyBasicsScreen Instead of Creating New Screens?

1. Consistency: Landlords adding a 2nd property will see the same flow they saw during onboarding. No learning curve.

2. Proven UX: PropertyBasicsScreen already handles address validation, property types, and bed/bath counts correctly.

3. Maintenance: One screen to maintain instead of duplicating logic across onboarding + property creation flows.

4. Airbnb Alignment: Airbnb collects property type, bedrooms, bathrooms, and guest capacity on their "property details" screen - same as PropertyBasicsScreen.

Why Reuse PropertyAreasScreen?

1. Smart Auto-Generation: Automatically creates "Master Bedroom", "Bedroom 2", "Bedroom 3", etc. based on bedroom count.

2. Property-Type Awareness: Shows different optional areas for apartments (Balcony, Laundry) vs houses (Garage, Yard).

3. No User Confusion: User doesn't have to manually select "Bedroom" 4 times - it's done automatically.

4. Airbnb Alignment: Similar to how Airbnb auto-structures listings based on property type and size.

Why Skip PropertyAssetsScreen During Onboarding?

1. Too Much Detail: Adding individual appliances/fixtures is overwhelming during initial setup.

2. Airbnb Pattern: Airbnb doesn't require detailed asset inventory during listing creation - they focus on amenities (high-level features).

3. Can Add Later: Landlords can add assets after onboarding from property details screen.

4. Faster Completion: Reduces drop-off by keeping onboarding focused on essentials.

What Stays NEW for Onboarding?

  • Welcome/Name/Account/Role: First-time user experience, account creation
  • Landlord Welcome + Property Intro: Set expectations, explain what's coming
  • Tenant Invite: Onboarding-specific - immediately connect tenant
  • Success Screen: Celebration, guide to dashboard
